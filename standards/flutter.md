When working on Flutter code:

- Use the project's existing state management solution (Riverpod, Bloc, Provider, etc.).
- Follow the Widget composition pattern. Keep build methods focused on layout, not logic.
- Use named routes or the project's existing navigation pattern.
- Separate business logic from UI widgets. Use the project's established architecture.
- Test widgets with widget tests, logic with unit tests.
- Follow Dart's effective naming conventions and linting rules.
