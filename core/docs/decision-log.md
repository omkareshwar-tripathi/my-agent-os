# Decision Log

Use this file to record reusable project decisions, tradeoffs, and assumptions. Keep it stack-neutral unless the target project explicitly needs a stack-specific decision.

## When To Add An Entry

- A choice has multiple reasonable options.
- A decision affects future work.
- The user approved a tradeoff.
- You are reversing or narrowing a previous choice.
- A constraint should prevent future agents from re-litigating the same issue.

## Entry Format

```markdown
## YYYY-MM-DD - Decision title

Context: What problem or ambiguity triggered the decision.

Options: The realistic choices considered.

Decision: What was chosen.

Reason: Why this choice fits the project now.

Consequences: What future agents must preserve, verify, or revisit.
```

## Open Decisions

Record unresolved questions here instead of silently guessing.
